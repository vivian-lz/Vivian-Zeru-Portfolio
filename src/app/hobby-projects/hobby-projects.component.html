    4-Bit ALU (Arithmetic Logic Unit)

  • This is an 4-bit ALU designed using two different methods: by using cases to build a 1-bit ALU and use that to scale up to a 4-bit ALU with interfaces and by utilizing the hardware structure of an ALU that is defined by a full adder and multiplexer. This ALU has a 2-bit opcode, or ALUControl that can compute Add, Subtract, AND, and OR functions. The add and subtract functions are created utilizing twos complement digital logic. This 4-bit ALU also includes a testbench that works for either design due to proper encapsulation.

  • While I have limited HDL and Verilog knowledge from digital systems coursework, I have been expanding my knowledge by using the hdlbits website to practice Verilog, as shown here: GitHub Repo.
  • This entire project is completed (including the created testbench).

  • The testbench was simulated in EDA Playground with Synopsys VCS, showcasing a 6.74% decrease in CPU time (runtime) for the structural design methodology (0.430 s) versus the behavorial methodology (0.460 s).

    •     Modified Weather Station (In Progress)

  • Designing a PCB in Eagle that uses an ESP32 and BME280 sensor to display the temperature, humidity, pressume, and approximate altitude on an LCD while using an RGB LED to visually display the temperature as well. trends.

    • Features
  • ESP32 IC acts as the main chip/microcontroller of this device with the BME280 IC creating the weather station capabilities.
  • Uses I2C communication between the microcontroller and the BME280
  • Powered through charging with a USB-C port that is powered on 5 V
  • Board runs on 3.3 V power
  • 5-3V LDO to regulate voltage between ESP32 and USB-C connector
  • Utilizes a crystal (40 kHz) as the main clock of the ESPS32 and the device itself.
  • Includes a reset button to manually prevent freezing issues with the ESP32 IC.
  • RGB LED displays 6 colors to visually represent the temperature, while an LCD display visually shows the user the temperature, humidity, pressure, and approximate altitude given by the BME280.
  • Firmware to run on the ESP32 microcontroller is created in C++ utilizing the Arduino IDE.
  • Schematic takes advantage of decoupling and load capacitors as well as pullup, pulldown, and current-limiting resistors, with a total of 13 of each.
  • Layout utilizes differential routing and matches the routing for high speed signals like the differential pairs for the USB-C and the I2C SDA/SCL lines.

    • Current Progress:
  • I created this in high school using an Arduino Uno. Now, I have made this project more advanced by designing the PCB in Eagle that uses an ESP32 instead.
  • I already completed the version that utilizes the ESP32 and BME280 modules, but not the raw ICs. I designed it in Eagle and have not yet ordered it through JLCPCB for manufacturing and to assemble the device, but this will be done in the future.
  • I am currently working on finishing the IC version, which is described below:
  • I have created the schematic for the board, utilizing an LDO, USB-C connector, USB to Serial Chip, ESP32 IC, BME280, Crystal to control the clock of the ESP32, reset button to control the ESP32 in case of freezing, an RGB LED as in the previous version, and an LCD as in the previous version to display the temperature, humidity, pressure, and approximate altitude.
  • I am currently working on the board, which right now is expected to result in a 24.73% decrease in area compared to the version utilizing the modules. (4401.71mm2 versus 3432.93mm2)
  • I am also in the process of learning to gain more control over the BME280 communication protocols by utilizing I2C communication through an STM32 Nucelo-64. In the future, I would like to use Verilog to create the sensor and control the communication even more. I also would hope to get the PCBs manufactured to assemble a full working product.
